ANGULAR: O que é?

O Angular é uma framework de estrutura web baseada em Typescript e atualmente está na versão 12. A sua estrutura é baseada em componentes e oferece maior desempenho para grandes e complexos projetos.

Esta ferramenta é de código aberto e atualmente é mantida pela Google, desta forma, está sempre em contínuo desenvolvimento.

O seu suporte é multiplataforma e é muito usado para o desenvolvimento front-end.

Estrutura Modular

Como dito anteriormente, o Angular é estruturado em módulos, mas o que isso quer dizer? Isso significa que todo o seu funcionamento é dividido e desenvolvido em partes, ou seja, dentro de cada projeto haverá:

  • Módulos (modules): é a parte de um todo, como um subsistema do app (parte principal);
  • Componentes (components): são os blocos de códigos responsáveis pela comunicação com o html;
  • Serviços (services): é onde colocamos toda a parte lógica, a 'ação' do código em si. Tem relação com os componentes.

Motivos para aprender Angular:

O Angular é muito usado e procurado nos desenvolvedores hoje em dia, pois o desempenho dos projetos são mais rápidos e fáceis de organizar. As empresas tem aplicado esta framework para grandes projetos e são poucos os programadores que tem domínio nela. Além do mais, o seu uso ajuda a dividir as tarefas entre a equipe, a encontrar os bugs mais facilmente e a reutilizar os códigos já feitos.

Desta forma, ensinaremos aqui o básico para que você consiga implementar o primeiro projeto. Siga os passos abaixo e em caso de dúvidas, vá para a seção de links úteis no rodapé.

Preparando o ambiente

1. Para instalar a framework, é necessário utilizar o Angular CLI (Command Line Interface), ao utilizar o -g, ele já será instalado globalmente no seu computador. Sendo assim, abra o prompt de comando e execute:

npm install -g @angular/cli

Obs: Caso queira consultar a versão, você pode digitar este comando:

ng --version

2. Entre na pasta escolhida (cd + nome da pasta) e execute o comando que vai criar o novo projeto. O nome do nosso projeto aqui é teste, mas você deve nomeá-lo como preferir.

ng new teste

Este comando vai criar toda a estrutura do seu projeto. Quando iniciar, será necessário escolher se quer já criar as rotas e o estilo inicial, por ora, escolha apenas N e CSS.

3. Acesse a pasta teste (cd + nome do projeto) e execute o comando que vai exercer a função de "servidor" do projeto, com ela você conseguirá acompanhar no navegador e no prompt as mudanças, falhas, etc.

ng serve

Após o prompt exibir "Compiled successfully", você pode acessar o endereço que ele sugerir, geralmente é colocado em: localhost:4200, mas caso esteja usando, ele poderá sugerir outras portas. Desta forma, a visualização deverá ser a seguinte:

4. Cada parte do projeto pode ser dividida em um módulo, por exemplo, módulo cliente, módulo funcionários, módulo produtos, etc. Para criar um módulo, você deve executar o comando desta forma:

ng g module testando

5. O módulo APP que é o principal, precisa saber da existência deste módulo novo, portanto, utilizando a técnica de barrels, você pode criar um arquivo index.ts dentro da pasta testando e exportar tudo do novo módulo. Após este passo, no arquivo app.module você deve importar o módulo novo. Lembre-se de repetir este processo a cada módulo novo. Também devemos repetir a cada nova pasta criada, pois os componentes precisam conversar entre si, sem a exportação e importação, eles não conseguem se enxergar e a aplicação não rodará.

6. Feito os passos acima, para criar um componente, você deve executar este comando:

ng g component testando/meus-componentes

Obs: os componentes sempre devem estar dentro do módulo, portanto atente-se em colocar primeiramente a pasta do módulo onde será criado e em seguida, o nome da pasta de componentes. Você pode colocar qualquer nome e depois renomear para 'components'. Faça os mesmos passos do index.ts, exportação e importação.

8. Como é pelo componente que é feita a comunicação com o html, devemos utilizar a sua tag (criada no arquivo .ts do component) para inserir no html principal, após apagar todo o conteúdo deste arquivo. Ex: <app-testando> </app-testando>

9. Os componentes são apenas as regras de negócio da aplicação, ainda precisamos criar as ações, portanto, execute este comando para criar um serviço:

ng g service testando/services/testar

Obs: Para criar os serviços, precisa criar a pasta services dentro do módulo e dar um nome ao serviço criado. Lembrando de novamente executar o processo de criação do index.ts, exportação e importação.

Estes são os passos básicos para montar a estrutura do seu projeto. Para preencher com o html e css, você deve utilizar de acordo com o que deseja, deixando no principal o que for padrão e comum no projeto, como estilo padrão, header e footer e em cada component pode preencher de acordo com a página. Ainda é possível trabalhar com rotas via Routers.

Diretivas

O Angular é repleto de componentes que auxiliam a desenvolver o código, entre estas opções, temos as diretivas. As diretivas nada mais são do que funcionalidades da própria framework, responsáveis por impor comportamento e manipular o DOM.

Ou seja, com as diretivas você pode criar comportamentos como se fosse via JavaScript e até mesmo criar a sua própria diretiva com o comando:

ng g directive testando/shared/teste-concluido

Obs: O ideal ao criar uma diretiva é que seja dentro de uma pasta compartilhada (shared). Não há limites de diretivas a serem usadas num projeto.

As diretivas mais conhecidas sãs estas a seguir:

  • *ngStyle: usada para alterar o estilo;
  • *ngClass: usada para alterar/inserir classes;
  • *ngIf: assim como o If de JS, é utilizada para condicionais;
  • *ngSwitch: da mesma forma que o If, serve para condicionais específicas;
  • *ngFor: também segue o princípio lógico de JS, utilizado para laços de repetição.

Data Binding

O databinding é outra funcionalidade do Angular, na qual permite a vinculação de dados. Exemplos:

  • { { } } - Interpolação (String Interpolation): geralmente usa o name do atributo ou alguma expressão em JS, como: { { title } }, para apresentar na tela o valor determinado, sem nenhuma alteração nele internamente.
  • [ ] - Propriedade Binding (Property Binding ou One Way Binding): com esta você consegue alterar o conteúdo da propriedade de acordo com que for passado, por exemplo: ( [ value ] = " Cancelar ")
  • ( ) - Evento (Event Binding): semelhante aos eventos de JS, esta funcionalidade serve para chamar os eventos sem o 'on' inicial, por exemplo: onclick → (click) ou onblur → (blur).
  • [ ( ) ] - Dois Caminhos (Two Way Binding): este serve para conectar o componente e o html ao mesmo tempo, ou seja, como uma via de duas mãos, muito usado para receber dados de formulários html. Para utilizá-lo em forms é necessário importar o módulo de formulário do Angular.

Pipes

Outra funcionalidade do Angular muito usada são os Pipes. Estes são filtros que servem como máscara para algumas aplicações. Para criá-lo o processo é bem parecido com os anteriores:

ng g pipe testando/pipes/data-teste

Obs: Assim como as diretivas, os pipes podem ser usados sem limite, mas o ideal é que sejam criados na mesma pasta 'pipes', para facilitar a organização.

Outras frameworks

Ainda no projeto de Angular, é possível usar Bootstrap e Jquery, basta fazer primeiramente a instalação via CMD e a importação dos módulos, conforme abaixo:

npm i --save jquery

npm i --save bootstrap@4

Obs: A versão que você escolher usar em seu projeto deve vir após o @, no caso aqui estamos usando a 4.

Após as instalações, abra o arquivo angular.json e insira: "node_modules/jquery/dist/jquery.min.js" (linha 37) e "node_modules/bootstrap/dist/css/bootstrap.min.css" (linha 34). Ao atualizar este arquivo, reinicie o server, pois o arquivo json só é lido uma vez a cada vez que rodar o código.
